# AGENTS.md

Cairn keeps its agent and contributor guidance in **one** place — **[`CLAUDE.md`](./CLAUDE.md)** at
the root, plus a per-directory `CLAUDE.md` for folder-scoped context.

**If your tool looks for `AGENTS.md`, read [`CLAUDE.md`](./CLAUDE.md) instead.** It holds the
build/test gate, the workspace map, and the conventions. This file is only a pointer, so there is a
single source of truth to maintain — not two copies to keep in sync.
